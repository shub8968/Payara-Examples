Realm-based Security Example

This example demonstrates a Jakarta EE Security based on realms. The realm used is `file`, one of the default realms defined by Payara. You can manage users using admin GUI.

You can try several pages, either unsecured or under some level of protection:

  • info – user info, displays user name and roles of the user
  • public.xhtml – public page
  • secured/users.xhtml – page protected by password, accessible to all users
  • secured/SecuredServlet – servlet protected by password, accessible to all users
  • secured-ssl/users.xhtml – page protected by password, accessible to all users, but only via secure HTTPS
  • admin/admins.xhtml – page protected by password, accessible only to all administrators. Regular users will get 403 - access denied.
  • logout – logout

REST resources:

  • rest/v1/public – public resource
  • rest/v1/protected – protected resource
  • rest/v1/admin – protected resource for admin users
  • rest/v1/public/me – information about logged-in user
  • rest/v1/security/login?username=Adam&password=Eve – login Adam
  • rest/v1/security/login?username=admin&password=5FsNqxmBug6x4eR – login Adam
  • rest/v1/security/logout – logout

These accounts are available:

  • user Adam in group user, password Eve
  • privileged user admin in groups admin and user, password 5FsNqxmBug6x4eR
